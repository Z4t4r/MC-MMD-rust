# Change: 添加骨骼可见性切换功能

## Why

当前 MMD 模型渲染时无法单独控制骨骼的可见性,导致调试和演示时不够灵活。用户希望能够切换特定骨骼的显示状态以便更好地理解模型结构。

## What Changes

- 在 Rust 引擎中添加骨骼可见性标记
- 通过 JNI 暴露骨骼可见性设置接口
- Java 渲染器根据可见性标记渲染骨骼调试视图
- 添加用户界面控制骨骼显示

### 用户界面变更

```ascii
+------------------------------------------+
| MMD 模型查看器                           |
+------------------------------------------+
| [模型预览区域]                           |
|                                          |
+------------------------------------------+
| 显示控制:                                |
| [X] 显示骨骼    [ ] 显示物理             |
| [X] 显示 IK 链   [ ] 显示刚体            |
| [应用到所有骨骼]  [重置]                 |
+------------------------------------------+
```

## Impact

### 受影响的规范
- **新增规范**:
  - `mmd-rendering` - MMD 渲染系统能力

### 受影响的代码

#### Rust 代码
- `rust_engine/src/model/runtime.rs` - 添加骨骼可见性标记
- `rust_engine/src/jni_bridge/model_handle.rs` - 添加设置接口

#### Java 代码
- `common/src/main/java/.../renderer/BoneRenderer.java` - 根据标记渲染
- `common/src/main/java/.../ui/DebugOverlay.java` - 添加 UI 控制

### 预期收益
- 更方便的模型调试
- 更好的演示效果
- 不影响性能（默认关闭）

### 性能影响
- **预期影响**: 无影响（默认关闭，开启时额外渲染开销 < 1ms）
- **测量方法**: 在渲染性能测试中测量
